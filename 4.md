## 4. 术语与定义

### 4.1. 术语

#### 4.1.1. 电力线载波通信模块

电力线载波通信（PLC）技术通过现有的 AC 电力传输线路进行数据传输。PLC 模块将数据调制到高频信号上，并利用电力线网络进行数据传递。这种技术广泛应用于智能电表、家庭自动化、远程监控和控制系统等领域。

PLC 模块，如 G3-PLC、PRIME 等，是典型的电力线通信技术代表，专门用于通过电力线进行数据传输。

这些通信模块通常用于集中器与采集器之间，或者集中器与电能表之间的数据通信。

接口说明：

- 接口 I1：主站系统与集中器之间的通信接口，由集中器部分定义；物理通信介质为 GPRS 等。

- 接口 I2：集中器与路由模块之间的通信接口，在模块与集中器的接口规范中定义；物理通信介质为 TTL 等。

- 接口 I3：路由模块与电表模块之间的通信接口，主要支持载波通信和RF通信，应用层数据采用标准的 DLMS/COSEM 数据格式。

- 接口 I4：模块与电表之间的通信接口，本规范的协议主要聚焦于此；物理通信介质为 TTL 等。

- 接口 I5：主站系统与网络通信模块（适用于GPRS、3G、4G、WIFI、Ethernet）连接的电能表之间的通信接口，采用标准的 DLMS/COSEM 数据格式。

### 5.1. 协议应用 (I3接口)

对于 G3-PLC 模块，需要支持 G3 分帧协议。具体而言，当传输给电表模块的数据帧长度超过物理层的限制时，模块应能够完整接收这些数据帧并进行分帧处理，然后将其打包发送给电表。相反，当电表发送的数据长度较长时，模块同样需要具备分帧处理的能力，以确保数据的完整传输。

当前 DLMS 数据帧的最大长度为 1250 字节。

### 5.2. 协议应用（I4接口）

本协议适用于表计与 PLC 及 RF 模块的串口通信。

#### 5.2.1. 物理接口参数

本协议采用串口通信，参数设置为 9600-8-N-1：波特率 9600，1 位起始位，8 位数据位，1 位停止位，无校验。

帧的基本单元为8位字节。链路层的传输顺序为高字节在前，低字节在后。

##### 5.2.3.2. 通讯模块与电表之间的心跳机制 (2002)

通讯模块与电表之间的心跳机制确保了两者连接的稳定性。模块会定期向电表发送心跳信号，如果电表在设定时间内（该时间由命令"2004"中的心跳周期定义）未接收到心跳信号，则认为模块发生异常。此时，电表将启动相应的处理机制，如事件记录和模块复位等操作。图2中的时序图提供了这一过程的直观展示。

• 定时发送：模块根据预设的时间间隔发送心跳信号，以确保与电表的通信持续有效。时间间隔默认 3 分钟。

• 状态反馈：电表在收到心跳信号后，能够及时确认模块的在线状态，并可根据需要进行相应处理。

异常处理：如果在预定时间内未收到心跳信号，电表会认为模块可能出现故障或断开连接，进将触发相应的处理机制，包括事件记录和模块复位等操作。

###### 5.2.3.6.4. DLMS数据帧封装 (1301)

由于模块与电表之间采用 UART 通信，为了防止外界干扰对 UART 数据传输的影响，需要将数据封装到本地协议中。本协议提供有效的数据校验机制，确保数据传输的可靠性和完整性。

数据封装选项：

此功能开关在“获取电表的信息(2004)-功能控制字”中配置和获取。

• 0 = 禁止：在此模式下，数据将以透传方式发送，不进行任何封装。

• 1 = 使能：此模式下，数据将被封装在本地协议中，以增加数据的完整性和安全性。

{

    while (len--)
    {
        fcs = (uint16_t)((fcs >> 8) ^ crc16Table [(fcs ^ *cp++) & 0xff]);

    return (fcs ^ 0xFFFF);
}

## 1 内部版本号定义

厂商代码、芯片代码、内部版本号每个部分均为两个字节。厂商代码、芯片代码、版本号均按逆序传输，如：V0102，报文中为 0201。主节点地址在报文中传输顺序为逆序，如地址：000000001234，报文中为 3412000000000。

### 1.1 HPLC 单模主节点

主节点程序包含 ST 和 3460。为与兼容现场已发货程序，不进行 ST 和 3460 间接口协议的扩展。

ST 程序包含应用层、网络层；已在详细信息里扩展 5 个字节的网络层版本信息；

3460 程序包含物理层、底层；物理层将原 MCU 版本日期改名称为版本号；底层的版本号定义见附录。

协议见附录 A。

### 1.2 HPLC 单模从节点

从节点只有 3460 芯片；分为 mcu 和 dsp 两部分程序。Mcu 部分的程序包含：应用层、网络层、物理库，为减少多种测试工具及各层的接口改动，分别将原扩展的内外部版本信息应答报文中名称进行变更。底层版本号见附录。

协议见附录 B。

### 1.3 HPLC 双模主从节点

HPLC 双模节点只有 3960 芯片；分为 mcu 和 dsp 两部分程序。Mcu 部分的程序包含：应用层、网络层、MAC 层、物理库、加密库、无线库等，为支持新增接口库，支持各层接口库改动，兼容硬件网省等相关信息，统一采用动态数据项 ID 的方案进行管理，采用模块端自维护方式，数据 ID 统一管理，方便新网省增加新库版本，方便新硬件增加新硬件特性，内部配置 ID 与升级匹配 ID 相结合，生产测试架与生产文件相结合方式，提高生产效率。

HPLC 双模主节点协议见附录 C.

HPLC 双模从节点协议见附录 D。

# 南网协议解析工具

## 项目概述

本项目是一个用于解析**中国南方电网有限责任公司计量自动化终端本地通信模块接口协议**（Q/CSG1209021-2019）的工具集。该工具支持对南网通信协议帧进行完整的解析、分析和生成，适用于低压电力线载波、微功率无线、双模、以太网等本地通信组网方式。

### 主要功能

- **单帧解析**：解析十六进制协议帧，展示详细的帧结构（帧头、长度域、控制域、地址域、AFN、SEQ、DI、数据内容、校验和、结束符）
- **批量解析**：支持批量解析多个协议帧，提供统计和导出功能
- **DI查询**：查询数据标识编码（DI）的含义，支持自定义DI管理
- **帧生成**：支持构建和打包协议帧
- **深化应用解析**：支持台区识别、交采数据、模块资产信息等深化应用协议

### 技术栈

- **Python 3.x**
- **PySide6**：GUI桌面应用框架
- **Streamlit**：Web应用框架
- **ctypes**：用于控制域等二进制结构体处理

## 核心组件

### 1. ProtocolFrameParser (`protocol_parser.py`)

核心解析器类，提供以下功能：

- **帧解析**：`parse(frame_bytes)` - 将字节帧解析为结构化字典
- **表格解析**：`parse_to_table(frame_bytes)` - 解析为表格数据（用于GUI展示）
- **DI映射**：内置完整的 DI_COMBINATION_MAP（DI3:DI2:DI1:DI0 → 业务说明）
- **AFN映射**：AFN（应用功能码）定义（0x00~0xF0）
- **自定义DI**：支持从 `custom_di.json` 加载用户自定义DI条目

**支持的AFN类型**：
- 0x00：确认/否认
- 0x01：初始化模块
- 0x02：管理任务
- 0x03：读参数
- 0x04：写参数
- 0x05：上报信息
- 0x06：请求信息
- 0x07：传输文件
- 0x10：维护命令（无线维护）
- 0x21~0x31：采集器接口
- 0xF0：维护模块（厂家自定义）

### 2. ControlField & Frame (`protocol_tool.py`)

- **ControlField**：控制域的二进制结构体（DIR、PRM、ADD、VER）
- **Frame**：帧数据结构，支持 `frame_pack()` 组帧功能

### 3. GUI应用 (`main_gui.py`)

基于PySide6的桌面应用，包含三个标签页：
- **单帧解析**：输入十六进制报文，展示解析结果表格
- **DI查询**：搜索和浏览DI映射表，支持添加/删除自定义DI
- **批量解析**：批量解析多帧报文，支持导出JSON结果

### 4. Web应用 (`streamlit_app.py`)

基于Streamlit的Web界面，提供：
- 报文输入和解析
- 完整JSON展示
- 帧结构可视化
- 原始字节分析

## 运行和构建

### 运行GUI应用

```bash
python main_gui.py
```

**依赖安装**：
```bash
pip install PySide6
```

### 运行Web应用

```bash
streamlit run streamlit_app.py
```

**依赖安装**：
```bash
pip install streamlit
```

### 打包为可执行文件

使用PyInstaller打包（配置文件：`南网协议解析工具.spec`）：

```bash
pyinstaller 南网协议解析工具.spec
```

生成的可执行文件位于 `dist/` 目录。

## 协议帧格式

### 帧结构

```
┌─────────┬──────────┬─────────┬──────────┬───────────┬─────────┬─────────┐
│ 起始符  │  长度L   │ 控制域C │ 地址域A  │ 应用数据  │ 校验和  │ 结束符  │
│ 1字节   │  2字节   │  1字节  │  12字节  │  变长     │  1字节  │  1字节  │
│  68H    │ 小端序   │         │ (可选)   │           │  CS     │   16H   │
└─────────┴──────────┴─────────┴──────────┴───────────┴─────────┴─────────┘
```

### 用户数据区结构

```
┌──────────┬─────────┬─────────┬─────────┬──────────────┐
│ 地址域 A │  AFN    │  SEQ    │   DI    │  数据内容    │
│ (可选)   │  1字节  │  1字节  │  4字节  │   变长       │
└──────────┴─────────┴─────────┴─────────┴──────────────┘
```

### 示例报文

**确认帧**：
```
68 0E 00 00 00 00 01 00 01 E8 00 05 EF 16
```

**复位硬件帧**：
```
68 0C 00 40 01 00 01 01 02 E8 2D 16
```

## 开发约定

### 代码风格

- Python 3.x，使用类型提示（typing模块）
- 类名使用CamelCase（如 `ProtocolFrameParser`）
- 函数/变量名使用snake_case（如 `parse_frame`）
- 常量使用UPPER_CASE（如 `AFN_MAP`、`DI_COMBINATION_MAP`）

### 数据结构

解析结果采用字典格式，结构化字段包含：
- `原始值`/`原始字节`：十六进制原始数据
- `解析值`/`十进制`：解析后的值
- `说明`/`名称`/`业务说明`：描述信息

### 自定义DI管理

自定义DI存储在 `custom_di.json` 文件中，格式：
```json
[
  {
    "di3": 232,
    "di2": 0,
    "di1": 8,
    "di0": 3,
    "desc": "查询采集任务号"
  }
]
```

通过以下方法管理：
- `ProtocolFrameParser.load_custom_di_list()`：加载自定义DI列表
- `ProtocolFrameParser.save_custom_di(custom_list)`：保存自定义DI列表

项目包含完整的协议标准文档：
- **2.pdf**：Q/CSG1209021-2019 原始PDF标准文档
- **1.md**：协议标准Markdown格式（帧结构、用户数据结构）

- **2.md、4.md**：其他协议相关文档

## 注意事项

1. **字节序**：长度域、DI等字段使用**小端序**（little-endian）
2. **地址格式**：地址为BIN格式，BCD格式需转换
3. **校验和**：控制域和用户数据区的八位位组算术和（不考虑溢出）
4. **传输方向**：DIR=0为下行（集中器→模块），DIR=1为上行（模块→集中器）
5. **协议版本**：当前协议版本VER=0
